# Frontend Prompt: Level Up (повышение уровня персонажа)

Нужно реализовать полный flow повышения уровня персонажа с поддержкой мультикласса, выбора наград и отображения результата.

## Суть фичи

1. Персонаж накапливает XP в ходе кампании.
2. Когда XP достигает порога, персонаж может повысить уровень.
3. Игрок (или GM кампании) открывает экран level-up, выбирает класс для повышения (мультикласс) и награды уровня.
4. Бэкенд применяет изменения: обновляет уровень, HP, hit dice, сохраняет выбранные награды.
5. Фронт показывает результат: что изменилось.

## Backend Endpoints

### Level Up

- `GET /api/characters/{id}/level-up-options` — получить опции повышения уровня
- `POST /api/characters/{id}/level-up` — применить повышение уровня
- `GET /api/characters/{id}/rewards` — посмотреть все полученные награды персонажа

### Существующие (для контекста)

- `GET /api/campaigns/{campaignId}/characters/{characterId}` — полные данные персонажа
- `GET /api/campaigns/{campaignId}/characters/{characterId}/stats` — характеристики

## Request / Response

### GET /api/characters/{id}/level-up-options

```json

    "xpToNextLevel": 0,

        "currentLevelInClass": 4,
        "newLevelInClass": 5,
        "rewardGroups": [
          {
            "rewardType": "FEAT",
            "isChoice": true,
            "rewards": [
              {
                "rewardEntryId": "uuid-entry-1",
                "rewardId": "uuid-feat-sentinel",
                "name": "Sentinel",
                "description": "Creatures provoke opportunity attacks...",
                "alreadyAcquired": false
              },
              {
                "rewardEntryId": "uuid-entry-2",
                "rewardId": "uuid-feat-grappler",
                "name": "Grappler",
                "description": "You have advantage on attack rolls...",
                "alreadyAcquired": true
              }
            ]
          },
          {
            "rewardType": "SKILL",
            "isChoice": false,
            "rewards": [
              {
                "rewardEntryId": "uuid-entry-3",
                "rewardId": "uuid-skill-athletics",
                "name": "Extra Attack",
                "description": "You can attack twice...",
                "alreadyAcquired": false
              }
            ]
          }
        ]
      },
      {
        "classId": "uuid-rogue",
        "className": "Rogue",
        "currentLevelInClass": 0,
        "newLevelInClass": 1,
        "rewardGroups": [
          {
            "rewardType": "SKILL",
            "isChoice": false,
            "rewards": [
              {
                "rewardEntryId": "uuid-entry-4",
                "rewardId": "uuid-skill-sneak-attack",
                "name": "Sneak Attack",
                "description": "Once per turn, extra damage...",
                "alreadyAcquired": false
              }
            ]
          },
          {
            "rewardType": "SUBCLASS",
            "isChoice": true,
            "rewards": []
          }
        ]
      }
    ]
  }
}
```

**Ключевые поля:**

- `currentTotalLevel` — текущий общий уровень персонажа (сумма всех классов)
- `xpToNextLevel` — сколько XP осталось до следующего уровня. `0` означает — можно повышать
- `availableClasses[]` — список классов, в которые персонаж может повысить уровень
  - Классы, которые уже есть у персонажа (`currentLevelInClass > 0`) — продолжить прокачку
  - Классы с `currentLevelInClass = 0` — взять мультикласс
- `rewardGroups[]` — награды, доступные на новом уровне класса
  - `isChoice: true` — игрок должен выбрать ОДИН вариант из списка `rewards`
  - `isChoice: false` — все награды этого типа получаются автоматически
  - `alreadyAcquired: true` — награда уже была получена ранее, показывать как недоступную

### POST /api/characters/{id}/level-up

Request body:

```json
{
  "classId": "uuid-fighter",
  "selections": [
    {
      "rewardType": "FEAT",
      "rewardEntryId": "uuid-entry-1"
    },
    {
      "rewardType": "ABILITY_SCORE_IMPROVEMENT",
      "rewardEntryId": "uuid-entry-asi-str"
    }
  ]
}
```

**Правила формирования `selections`:**

- `classId` — UUID класса, в котором повышается уровень. Должен быть из `availableClasses`
- `selections` — массив выбранных наград. Заполняется ТОЛЬКО для `rewardGroups` где `isChoice: true`
- Для `isChoice: false` — не нужно отправлять ничего, награды применяются автоматически
- `rewardEntryId` — это `rewardEntryId` из опций (НЕ `rewardId`)
- Нельзя выбирать награды с `alreadyAcquired: true`

Response: `ApiResponse<LevelUpResultResponse>`

```json
{
  "success": true,
  "message": "Уровень повышен",
  "data": {
    "newTotalLevel": 5,
    "classLeveled": "Fighter",
    "newClassLevel": 5,
    "hpIncrease": 8,
    "newMaxHp": 44,
    "rewardsAcquired": [
      { "rewardType": "FEAT", "name": "Sentinel" },
      { "rewardType": "SKILL", "name": "Extra Attack" }
    ]
  }
}
```

**Поля результата:**

- `newTotalLevel` — новый общий уровень
- `classLeveled` — имя класса, в котором повысился уровень
- `newClassLevel` — новый уровень в этом классе
- `hpIncrease` — на сколько увеличились HP (hitDie/2 + 1 + CON modifier, минимум 1)
- `newMaxHp` — новый максимум HP
- `rewardsAcquired` — список всех полученных наград (и автоматических, и выбранных)

### GET /api/characters/{id}/rewards

Response: `ApiResponse<CharacterRewardsResponse>` — все полученные награды, сгруппированные по классам.

```json
{
  "success": true,
  "data": {
    "characterId": "uuid",
    "totalLevel": 5,
    "classBreakdown": [
      {
        "classId": "uuid-fighter",
        "className": "Fighter",
        "classLevel": 5,
        "subclass": {
          "name": "Champion",
          "description": "..."
        },
        "rewardsByType": {
          "FEAT": [
            { "name": "Sentinel", "acquiredAt": "2026-06-01T12:00:00Z" }
          ],
          "SKILL": [
            { "name": "Extra Attack", "acquiredAt": "2026-06-05T18:00:00Z" }
          ]
        }
      }
    ]
  }
}
```

## Типы наград (RewardType)

| Тип | Описание | Поведение |
|-----|----------|-----------|
| `FEAT` | Черта (Feat) | Обычно `isChoice: true` — игрок выбирает одну черту из списка |
| `SKILL` | Классовая способность | Обычно `isChoice: false` — получается автоматически |
| `SUBCLASS` | Выбор подкласса | `isChoice: true`, можно выбрать только один раз на класс. Если подкласс уже выбран, группа не приходит |
| `BUFF_DEBUFF` | Пассивный бафф/дебафф | Может быть `isChoice: true` или `false` |
| `ABILITY_SCORE_IMPROVEMENT` | Увеличение характеристики | `isChoice: true` — игрок выбирает какую характеристику увеличить на 1 (можно выбрать дважды одну или две разных) |

## UI/UX Flow

### Шаг 0: Индикатор готовности

На странице персонажа показывать индикатор, если персонаж готов к повышению уровня.

Определение: вызвать `GET /api/characters/{id}/level-up-options`. Если ответ успешный и `xpToNextLevel === 0` — персонаж готов. Если бэкенд вернул ошибку (уже макс. уровень или не хватает XP) — не показывать кнопку.

```
┌─────────────────────────────────────────────────────┐
│  ⚔️ Торин Дубощит                                  │
│  Воин 4 • Дварф • HP: 36/36                        │
│                                                     │
│  XP: 6500 / 6500              [⬆ Повысить уровень] │
│  ████████████████████████████████ 100%               │
└─────────────────────────────────────────────────────┘
```

XP прогресс-бар: `(currentXP - prevThreshold) / (nextThreshold - prevThreshold) * 100%`

### Шаг 1: Выбор класса

После нажатия **"Повысить уровень"** → `GET /api/characters/{id}/level-up-options`.

Показать список `availableClasses`. Классы с `currentLevelInClass > 0` — "Продолжить", с `currentLevelInClass === 0` — "Мультикласс (новый)".

```
┌─────────────────────────────────────────────────────┐
│  Повышение уровня                         [Отмена]  │
│  Текущий уровень: 4 → 5                            │
│                                                     │
│  Выберите класс для повышения:                      │
│                                                     │
│  ┌─────────────────────────────────────────────────┐ │
│  │ ⚔️ Fighter (Воин)                               │ │
│  │ Текущий уровень: 4 → 5                          │ │
│  │ Награды: Extra Attack, Feat                      │ │
│  │                              [Выбрать]           │ │
│  └─────────────────────────────────────────────────┘ │
│                                                     │
│  ┌─────────────────────────────────────────────────┐ │
│  │ 🗡️ Rogue (Плут) — МУЛЬТИКЛАСС                   │ │
│  │ Новый класс: уровень 1                          │ │
│  │ Награды: Sneak Attack, Expertise                 │ │
│  │                              [Выбрать]           │ │
│  └─────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

В каждой карточке класса кратко показать, какие `rewardGroups` ждут (по `name` первой награды каждой группы).

### Шаг 2: Выбор наград

После выбора класса показать его `rewardGroups`.

Для каждой группы:

- **`isChoice: false`** — показать как "Автоматические награды" (серым, без выбора):
  ```
  ┌───────────────────────────────────────┐
  │ ✅ Автоматически                      │
  │                                       │
  │ • Extra Attack — You can attack twice │
  │ • Martial Versatility                 │
  └───────────────────────────────────────┘
  ```

- **`isChoice: true`** — показать как radio-группу или карточки для выбора:
  ```
  ┌───────────────────────────────────────┐
  │ 🎯 Выберите черту (FEAT)              │
  │                                       │
  │ ○ Sentinel                            │
  │   Creatures provoke opportunity       │
  │   attacks even if they Disengage...   │
  │                                       │
  │ ● Great Weapon Master     ← выбрано   │
  │   Before you make a melee attack...   │
  │                                       │
  │ ○ Grappler (уже получена)  ← disabled │
  └───────────────────────────────────────┘
  ```

- **`ABILITY_SCORE_IMPROVEMENT`** — особый UI. Каждый `rewards[]` entry — это одна характеристика. Игрок может выбрать одну (увеличить на 2) или две разных (каждую на 1). Обычно приходит 6 entries (по одному на каждую базовую характеристику). Реализация:
  ```
  ┌───────────────────────────────────────┐
  │ 📊 Увеличение характеристики (ASI)    │
  │                                       │
  │ Распределите 2 очка:                  │
  │                                       │
  │ Strength     14  [+] [-]  +1          │
  │ Dexterity    12  [ ] [ ]   0          │
  │ Constitution 13  [+] [-]  +1          │
  │ Intelligence 10  [ ] [ ]   0          │
  │ Wisdom        8  [ ] [ ]   0          │
  │ Charisma     15  [ ] [ ]   0          │
  │                                       │
  │ Осталось очков: 0                     │
  └───────────────────────────────────────┘
  ```
  В `selections` отправляются 1 или 2 записи с `rewardEntryId` выбранных характеристик. Если игрок вложил оба очка в одну характеристику — отправляется одна запись (бэкенд применит +1, можно отправить дважды одну запись для +2).

## Роутинг

```
/campaigns/:campaignId/characters/:characterId/level-up  → Wizard повышения уровня
/campaigns/:campaignId/characters/:characterId/rewards    → Просмотр всех наград
```

Кнопка "Повысить уровень" на странице персонажа ведёт на `/level-up`.

## Доступ

- **Игрок** — может повышать уровень ТОЛЬКО своего персонажа
- **GM кампании** — может повышать уровень любого персонажа в своей кампании
- **Админ** — может повышать уровень любого персонажа

Если у пользователя нет прав — бэкенд вернёт 403. На фронте скрывать кнопку "Повысить уровень" для чужих персонажей (исключение: GM видит кнопку для всех персонажей своей кампании).

## Мультикласс: визуальные подсказки

Если персонаж уже имеет несколько классов, показать их все в header:

```
Fighter 4 / Rogue 1 — Уровень 5
```

В карточках выбора класса (шаг 1) визуально отделять:
- **Существующие классы** (`currentLevelInClass > 0`) — показывать первыми, с иконкой прогрессии
- **Новые классы** (`currentLevelInClass === 0`) — показывать ниже, с пометкой "Мультикласс"

## Состояния и edge cases

### Персонаж уже 20 уровня

Бэкенд вернёт 400: `"Персонаж уже достиг максимального уровня"`. Скрыть кнопку level-up. Можно показать бейдж "MAX".

### Не хватает XP

Бэкенд вернёт ошибку. На фронте показать: сколько XP нужно (`xpToNextLevel`), сколько есть. Кнопка level-up неактивна (disabled).

### Нет наград на уровне

`rewardGroups` может быть пустым массивом. Показать: "На этом уровне нет выбираемых наград, HP и уровень будут повышены автоматически". Всё ещё нужна кнопка "Подтвердить".

### Подкласс уже выбран

Если у персонажа уже есть подкласс для данного класса, `SUBCLASS` группа НЕ приходит в `rewardGroups`. Ничего не делать.

### Все награды в choice-группе уже получены

Все `rewards` в группе имеют `alreadyAcquired: true`. Показать группу как "Все варианты уже получены", без возможности выбора.

## Error Handling

| Код | Когда | Сообщение |
|-----|-------|-----------|
| 400 | Максимальный уровень | "Персонаж уже достиг максимального уровня" |
| 400 | Класс недоступен в кампании | "Этот класс недоступен в текущей кампании" |
| 403 | Нет прав | "Нет прав на повышение уровня этого персонажа" |
| 404 | Персонаж не найден | "Персонаж не найден" |
| 422 | Не выбрана награда | "Не выбран вариант для типа награды: FEAT" |
| 422 | Награда уже получена | "Награда уже получена: uuid" |
| 422 | Уровень класса > 20 | "Уровень класса не может быть выше 20" |
| 422 | Подкласс уже есть | "У персонажа уже есть подкласс для этого класса" |

## Важные моменты

1. **`rewardEntryId` vs `rewardId`**: в `selections` отправляется `rewardEntryId` (идентификатор записи в таблице наград), НЕ `rewardId` (идентификатор самой сущности — черты, навыка и т.д.).
2. **HP обновляется автоматически** бэкендом. Не нужно отправлять HP в запросе. Результат приходит в `hpIncrease` и `newMaxHp`.
3. **Hit dice** обновляются автоматически. Формат строки: `"3d10"` (один класс) или `"3d10 + 2d8"` (мультикласс). Видно в `CharacterResponse.hitDiceTotal` после обновления.
4. **Мультикласс**: персонаж может повышать уровень в любом доступном классе. Общий уровень = сумма уровней всех классов. Максимум: 20.
5. **GM может повышать уровень** любого персонажа в своей кампании. Показывай кнопку level-up для GM.
6. **После level-up** обязательно обнови данные персонажа на фронте — HP, уровень, classLevels, hitDiceTotal, все изменились.
